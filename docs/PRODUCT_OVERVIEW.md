# UniHR 產品介紹

> 本文件適合非技術背景的讀者閱讀，包含產品簡介、核心功能、所有頁面說明，以及開發歷程。

## 一、產品簡介

**UniHR** 是一套專為臺灣企業設計的 **AI 人資助理 SaaS 平台**。

員工只需用自然語言提問，系統便會同時搜尋「公司內部規章」與「臺灣勞動法規」，合併產出最佳解答，並標明來源出處。企業不需要自行建置 AI 基礎設施，開通帳號即可使用。

### 解決什麼問題？

| 傳統方式 | UniHR 方式 |
|----------|------------|
| HR 人員接聽大量重複諮詢 | 員工自助查詢，AI 即時回答 |
| 翻閱紙本或 PDF 規章 | 上傳文件後全文可搜尋 |
| 不清楚勞基法規定 | 同步查詢勞動法 AI 引擎 |
| 各公司自建 IT 系統 | SaaS 多租戶，開箱即用 |

### 服務架構（雙層設計）

1. **Core 層（勞動法 AI 引擎）**：持續優化的臺灣勞動法問答知識庫，由平台統一維護。
2. **SaaS 層（本平台）**：每家企業擁有獨立空間，管理自己的帳號、文件、使用記錄與配額。

每次提問時，系統同時搜尋「您公司的規章」與「勞動法 Core」，合併回答。

## 二、核心功能

### 🏢 多租戶企業隔離
- 每家企業擁有完全獨立的資料空間，企業之間資料零交叉
- 可設定文件數量、每月查詢次數、儲存空間等配額上限
- 支援**白標品牌客製化**：上傳公司 Logo、設定主題色彩與企業名稱，打造專屬外觀

### 👥 帳號與角色管理
系統提供六個角色層級，依職責分配不同權限：

| 角色 | 說明 |
|------|------|
| 超級管理員（Superadmin） | 平台級別管理，管理所有企業租戶 |
| 擁有者（Owner） | 企業最高管理者，管理公司設定與訂閱 |
| 管理員（Admin） | 管理員工帳號、部門、文件與用量 |
| 人資管理員（HR） | 管理部門結構、上傳文件、查看 RAG 品質儀表板 |
| 員工（Employee） | 一般使用者，可使用 AI 問答與查閱文件 |
| 檢視者（Viewer） | 唯讀權限 |

- 支援 **Google / Microsoft 單一登入（SSO）**，以 Email 自動識別所屬企業
- 管理角色強制啟用**雙重驗證（MFA/TOTP）**，保護高權限操作
- 支援以邀請連結方式新增員工，無需手動建立帳號

### 📄 企業知識庫
- 上傳公司內規、HR 政策文件，系統自動處理並建立可搜尋的知識庫
- 支援 **23 種檔案格式**：PDF（文字型 / 掃描型）、Word（DOCX/DOC）、Excel、PowerPoint、CSV、圖片、HTML、Markdown、RTF、JSON、網頁 URL 等
- 文件上傳後自動評估解析品質（優/良/可/差/失敗），確保知識庫品質
- 可依部門分類管理文件

### 🤖 AI 問答
- 員工輸入問題，AI 即時搜尋知識庫並生成回答
- **打字機串流效果**：回答即時逐字顯示，不需等待
- **來源引用面板**：每個回答都附上引用來源，區分「公司政策」與「勞動法規」，含相似度評分
- **Markdown 格式渲染**：支援表格、清單、程式碼等結構化顯示
- **後續建議**：AI 自動生成相關追問，引導深入探索
- **對話歷史**：可搜尋過去的對話，並匯出為文字檔
- **回饋機制**：員工可對每個回答按讚👍或按爛👎，協助持續優化

### 🔍 智慧檢索引擎
- 同時使用「語意理解」與「關鍵字比對」兩種方式搜尋，取長補短
- 中文分詞精準處理，支援「NT$850,000」等格式
- 當主要向量資料庫異常時，自動切換備用搜尋，確保服務不中斷

### 📊 用量統計與分析
- 每位員工與整體公司的 AI 查詢次數、Token 使用量一目了然
- 管理員可查看熱門問題、部門統計、每日趨勢
- 超過配額時自動告警

### 📋 稽核日誌（合規用途）
- 完整記錄每次問答：誰問了什麼、引用了哪些來源、AI 如何回答
- 記錄**不可竄改**：每筆資料計算數位指紋，資料庫層阻擋任何修改或刪除
- 預設保存 **7 年**，滿足企業合規需求

### 💳 訂閱與金流
- 彈性訂閱方案，可依需求升級或降級
- 支援**藍新金流（NewebPay）**與 **Stripe** 付款
- 完整帳單記錄，可匯出

### 🌐 自訂網域與多地區
- 企業可設定自有網域（如 `hr.yourcompany.com`），搭配 SSL 自動申請
- 支援 4 個部署區域（亞太/北美/歐洲/日本），滿足資料駐留合規要求（PDPA / GDPR / SOC 2 / APPI）

### 🔒 資訊安全
- 登入憑證存於 HttpOnly Cookie，JavaScript 無法讀取，防範竊取攻擊
- CSRF 雙重驗證防護，防止跨站請求偽造
- 病毒掃描：所有上傳文件均經 ClamAV 掃描
- 2026-03-23 安全評估得分：**A- / 8.8 分**

## 四、開發計畫

本產品採用分階段迭代方式開發，共規劃 **13 個開發階段**，預計總開發時程約 **34 週**。

| 階段 | 主要內容 | 預估時程 |
|------|----------|----------|
| **Phase 1** | 核心基礎：多租戶架構、帳號認證、文件上傳管線、AI 問答基礎 | Week 1–3 |
| **Phase 2** | 安全強化：Google/Microsoft SSO 登入、速率限制、API 版本管理、功能旗標 | Week 4–5 |
| **Phase 3** | 管理功能：配額管理、用量分析、前端管理頁面 | Week 6–7 |
| **Phase 3+** | 文件引擎升級：支援 19 種格式、進階混合搜尋（語意 + 關鍵字） | Week 8–9 |
| **Phase 4** | 生產化：前後台分離、CI/CD 自動部署、白標品牌、監控系統、微服務架構、多地區支援（22 項任務） | Week 10–13 |
| **Phase 5** | 使用者體驗全面審查：11 項 UX 修復（路由守衛、SSO 自動識別企業、權限統一、UI 細節優化） | Week 14–15 |
| **Phase 6** | AI 引擎升級：LlamaParse 智慧解析、中文分詞（jieba）、HyDE 查詢擴展、更精準的 LLM 答案生成、重複內容去除 | Week 16–18 |
| **Phase 7** | 對話體驗升級：串流打字機效果、Markdown 渲染、來源引用面板、回饋系統、RAG 品質儀表板、行動版響應式（15 項任務） | Week 19–21 |
| **Phase 8** | 生產安全加固：啟動前 13 項安全硬性檢查、一鍵密鑰生成工具、生產容器安全強化（15 項安全特性） | Week 22–23 |
| **Phase 9** | 縱深防禦：Redis 熔斷保護、不可竄改稽核日誌、資料庫層 Row-Level Security、備份自動驗證、安全監控告警、壓力測試（7 項） | Week 24–25 |
| **Phase 10** | 資安全面強化：HttpOnly Cookie session（取代 localStorage）、CSRF 雙重驗證、SSRF 攻擊封鎖、SSO 安全白名單、管理角色強制 MFA（11 項） | Week 26–27 |
| **Phase 11** | 檢索引擎強化：向量儲存雙備援（Pinecone + PostgreSQL）、零品質向量過濾、BM25 快取優化、智慧解析策略調整 | Week 28–29 |
| **Phase 12** | AI 可觀測性：Langfuse 全鏈路追蹤（解析→嵌入→問答）、MCP Server（外部 AI 整合介面） | Week 30–31 |
| **Phase 13** | 品質儀表板、進度追蹤、品牌驗證、LLM 安全護欄（防止 Prompt 注入）、金流整合（藍新金流 + Stripe）、Email 服務 | Week 32–34 |

### 各階段重要里程碑

```
Phase 1–3   基礎平台建立，可運行最小可行產品（MVP）
Phase 4–5   生產化與用戶體驗完善，準備正式上線
Phase 6–7   AI 問答品質大幅提升，對話體驗現代化
Phase 8–10  全面資安強化，目標通過安全評估（A 級）
Phase 11–13 檢索精準度、可觀測性、金流、品質儀表板完善
```
